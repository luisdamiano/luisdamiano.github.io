Deep GPs: summary of Deep Gaussian Processes by Damiano & Lawerence (2013)

This is a quick, math-free summary of Deep Gaussian Processes by Damiano & Lawrence (2013). Many details are purposedly left out to lighten the read, see the full paper.

Disclaimer: As this is a first draft that has not been fully edited yet, watch out for errors, inaccuracies, and unfortunate choices of technical vocabulary. Let me know if you find any of these and I’ll get them fixed.

About the article

  • Main idea: introduce GP Gaussian Processes for learning tasks.
  • Motivation:
    • Probabilistic, yay! Know your uncertainty.
    • Deep model seem to better capture abstract information. Layers mimic inductive reasoning and the human concept of generalization.
  • Challenge: Can Deep GPs work with scarse data, where questions of generalization arise?
  • Contributions:
    • Exploit advances in variational inference to marginalize the latent variables in the hierarchy variationally. Show that any number of GPs can be stacked.
    • Use the strict lower bound on the likelihood given by variational inference for model selection in the context of scarse data.

Deep GP: big picture

  • Model: based on a deep belief network.
    • Structure: cascade of hidden layers of latent variables.
    • Output: data is modeled as the output of a multivariate GP (leaves of the hierarchy).
    • Inputs: governed by another GP.
    • Mapping: GPs.
    • Layers:
      • Single layer:
        • Equivalent to standard (read vanilla) GP if inputs are observed (e.g. regression, supervised learning) and fed to the upper most layer.
        • Equivalent to GP-LVM if inputs are treated as latent (e.g. unsupervised learning) and marginalized out.
          • Inference: as marginalization is analytically intractable, we maximize w.r.t. the latent variables instead of the parameters that will later be marginalized.
      • Multiple layers.
    • Inference: approximate variational marginalization.
    • Fully Bayesian inference (useful with both large and small datasets).
    • Model selection:
    • Tuning parameters: number of layers (depth), number of nodes per layer (width).
    • Quantity: strict lower bound on the marginal likelihood computed by variational marginalization.
    • Selection of size of the latent space:
    • Neal (1996): using Automatic Relevance Determination.
    • Damianou et al. (2011): placing a GP prior over the latent space (Bayesian Dinamical GP-LVM).
    • Damianou et al. (2013): approximately marginalizing any number of hidden layers.

Relationships with models in the GP family.

  • Standard GP:
    • Setting: regression and supervised learning.
    • Data: set of input-output pairs.
    • Model: unobserved, latent function generates the outputs given the inputs.
    • Inference:
      • Goal: estimate an unobserved, latent function.
      • Priors:
        • The GP is a nonparametric prior distribution over the latent function.
        • The choice of covariance function defines the properties of the latent mapping.
      • Likelihood: assuming that the collection of latent function instantiations is normally distributed, the likelihood of the outputs given the inputs (marginalizing out for the latent functions) can be computed analytically.
  • GP Latent Variable Model (GPLVM):
    • Setting: unsupervised learning.
    • Data: set of outputs.
    • Model: unobserved, latent function generates the outputs given unobserved, latent inputs.
    • Inference:
      • Goal: estimate an unobserved, latent function.
      • Priors:
        • Independent GPs as prior distribution over the latent function.
        • Priors over the latent variables?

Deep Gaussian Processes

The article explore Deep Gaussian Processes for unsupervised learning in more detailed, but claims it is true for regression and supervised learning

  • Setting: regression, supervised and unsupervised learning.
  • Data: set of input-output pairs.
  • Model: graphical model with three kind of nodes.
    • Leaf nodes: observed outputs.
    • Intermediate latent space.
      • Extended vertically by adding layers.
      • Extended horizontally by partitioning the output space of each layer.
    • Parent nodes:
    • Unobserved, potentially constrained with priors (e.g. dynamical priors) for unsupervised learning.
    • Observed, given inputs for regression or supervised learning.
  • Inference:
    • Challenges:
    • Each layers adds a significant number of model parameters.
    • Large regularization challenge.
    • Proposed method: marginalize out the whole latent space.
    • Motivation:
    • Occam’s razor.
    • Lower the number of model parameters.
    • Implementation:
      • Define an automatic relevance determination covariance function with a different weight for each latent dimension.
      • Regularization: use Bayesian inference to switch-off irrelevant dimensions in the latent space.
      • Bayesian inference is hard due to nonlinearities in the covariance function (when is Bayesian inference easy these days, anyway?).
